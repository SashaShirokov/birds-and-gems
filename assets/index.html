Rules

Level 1: Your goal is to collect any 10 gems within 1 minute.

Level 2: Your goal is to collect 8 gems the bird is thinking about. The wrong gem reduces your time bar by 5 seconds and the right gem adds 3 seconds.

Level 3: Your goal is to collect 7 gems the bird is thinking about. On this level the wrong gem reduces your time by 3 seconds, the right one adds 5 seconds and monsters take out 7 seconds.

Control

The gravity is applied to the bird that always pulls it down. Whenever you click the left mouse button you partially take the gravity off and make the bird fly. Play around and you will get accustomed to it. Enjoy your game!

Here’s how the game is built…

From the phaser we download just Phaser.min.js file. When we declare a new Phaser game we can use three modes: Phaser.WebGL, Phaser.Canvas and Phaser.Auto (this mode we used in the game). And in the same function we add all the states we're going to have.

In the state itself we allways use three main functions. The preload() function is where we load all images and sounds in our library. In create() function we use the library. This is where we set up objects, variables, sounds, text. It is here all parts of the game come together: good guys, bad guys, explosions etc.. And usually we position it here as well. The update() function is a constant running loop, and we use that to check things like collisions, movements and all sorts of things like that.

In the following resource openGameArt you can find a lot of wonderful assets of sounds and images. Most of them are free to use even in commercial games, but check the licence first.

Our Workflow

Gulp Automation

Gulp is at the heart of all automation that is set up in this project. Since gulp itself does not do anything it is lightweight, efficient and runs quickly. There is a gulp plugin to automate almost any task you can think of. We use three key tools of gulp: gulp.src() gulp.dest() and pipe(). And of course gulp.watch that makes automation possible.

CSS Organization

We follow three key concepts in order to write super organized CSS: 1. File Architecture - we break our css code into multiple files and each file is relativly small and has a specific purpose. 2. Identify patterns in design. 3. We have the rules to follow for creating class names and writing our css selectors - B.E.M. methodology.

PostCSS

We utilize a new post-processor that lets us to do some of the following things in css. For example, we can save a —Åertain color into a variable and reuse it throughout our css. We also can write nested css. This tool passes our code through filter that creates a new file where the code is automaticly converted into regular css that will work in a web browser.

Webpack and Babel

In JS organization we use webpack to bundle all our individual files together into a single file and that bundled file will work in a web browser. And with the help of Babel we are able to write modern JS and to have excellent browser support at the same time.
